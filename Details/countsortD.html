    Counting sort, also known as counting sort algorithm, is a non-comparative integer sorting algorithm. It works by counting the number of elements that have distinct key values and using these counts to determine the positions of the elements in the sorted output. Counting sort is efficient when sorting a range of integers with a small, known range of values. Here are the details of how counting sort works:

    1.Initialization : Determine the range of input values and create an auxiliary array, called the "counting array," to hold the count of each unique element within that range. The size of the counting array is typically set to cover the entire range of input values.
    2.Counting : Iterate through the input list, counting the number of occurrences of each distinct element. This process fills the counting array with the counts of each unique element.
    3.Cumulative Sum : Modify the counting array to store the cumulative sum of counts. This cumulative sum represents the position in the sorted output where each unique element should start.
    4.Sorted Output : Create an output array of the same size as the input list to hold the sorted elements. Iterate through the input list again, and for each element, look up its count and position in the cumulative sum array. Place the element in the output array at the specified position, decrement the count, and update the cumulative sum.
    5.Repeating Elements : If there are duplicate elements in the input list, this process ensures that they are placed in the correct order in the output array.
    6.Termination : The algorithm terminates when all elements from the input list have been placed in the correct positions in the output array.

    Counting sort has the following characteristics :

    • Time Complexity : Counting sort has a linear time complexity of O(n + k), where n is the number of elements in the list and k is the range of input values. This makes it very efficient when sorting integers with a small range.
    • Stability : Counting sort is a stable sorting algorithm, meaning it preserves the relative order of equal elements.
    • In-Place : Counting sort is not an in-place sorting algorithm because it requires the use of additional memory for the counting array and the output array.
    • Limited Applicability : Counting sort is not suitable for sorting real numbers or other data types with a large range of values, as it requires a counting array of equal size to the range of values.

    Counting sort is often used as a building block in more complex sorting algorithms or in situations where sorting integers with a small, known range is required, as it can provide efficient performance in such cases.
